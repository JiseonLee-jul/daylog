# Claude Code의 숨겨진 강력한 기능들 15가지

> 출처: https://news.hada.io/topic?id=28021
> 클로드 코드 제작자 Boris Cherny가 정리한 목록

## 1. 모바일 앱

iOS/Android 앱의 Code 탭에서 노트북을 열지 않고도 코드 변경 작업을 수행할 수 있다.

## 2. 세션 이동 (Teleport)

`claude --teleport` 또는 `/teleport` 명령으로 클라우드 세션을 로컬 머신에서 이어서 실행할 수 있다. `/remote-control` 명령으로 원격 제어도 가능.

## 3. /loop 및 /schedule

최대 1주일 단위로 자동 실행 스케줄을 설정할 수 있다. PR 관리, 코드 리뷰, Slack 피드백 등을 완전 자동화하는 데 활용 가능.

## 4. Hooks

에이전트 라이프사이클의 특정 시점에 커스텀 로직을 삽입할 수 있는 기능. 예를 들어 파일 저장 전/후에 린트를 자동 실행하는 식.

## 5. Dispatch

원격 제어 기능. 다른 머신이나 환경에서 실행 중인 Claude Code 세션을 제어할 수 있다.

## 6. Chrome 확장

프론트엔드 작업을 할 때 브라우저와 Claude Code를 연동해서 실시간으로 UI를 확인하며 작업할 수 있다.

## 7. 세션 포크 (/branch)

`/branch` 명령으로 현재 세션을 분기해서 여러 접근 방식을 동시에 시도할 수 있다. 실험적 변경을 안전하게 테스트하는 데 유용.

## 8. /btw

메인 작업 흐름을 중단하지 않고 사이드 쿼리를 던질 수 있는 기능. 작업 중 갑자기 궁금한 게 생겼을 때 맥락을 잃지 않고 질문 가능.

## 9. Git Worktrees

여러 브랜치를 동시에 체크아웃해서 병렬 작업을 지원한다. 하나의 레포에서 여러 기능을 동시에 개발할 때 유용.

## 10. /batch

수십~수천 개의 워크트리 에이전트에 작업을 분산 처리할 수 있다. 대규모 리팩토링이나 마이그레이션 같은 반복 작업에 적합.

## 11. --bare 플래그

SDK 시작 속도를 최대 10배 향상시키는 경량 모드. 자동화 파이프라인이나 스크립트에서 Claude Code를 빠르게 호출할 때 유용.

## 12. --add-dir

다중 저장소에 동시 접근할 수 있게 해주는 플래그. 모노레포가 아닌 환경에서 여러 관련 프로젝트를 함께 다룰 때 사용.

## 13. 커스텀 에이전트

`.claude/agents` 디렉토리에 에이전트 정의 파일을 두면 프로젝트에 특화된 에이전트를 만들 수 있다. 반복적인 워크플로를 에이전트로 캡슐화하는 방식.

## 14. /voice

음성 입력을 지원한다. 타이핑 대신 말로 지시를 내릴 수 있어서, 손이 자유롭지 않을 때 유용.

## 15. 기타

위 기능들은 Claude Code가 단순 코드 생성 도구가 아니라, 개발 워크플로 전체를 자동화하는 플랫폼으로 진화하고 있음을 보여준다.

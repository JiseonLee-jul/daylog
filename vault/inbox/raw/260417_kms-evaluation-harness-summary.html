KMS · Design Summary

Evaluation Harness

Author 이지선  ·  Date 2026-04-17  ·  Draft 팀 공유용
01 목적
KMS는 단번의 설계로 완성되지 않는다. 설계하고, 실 데이터에 부딪히고, 문제를 발견하고, 컴포넌트를 뜯어 고치는 과정이 반복될 수밖에 없다. 따라서 핵심은 문제를 빠르게 판단하고, 원인을 분석하고, 개선을 검증하는 루프를 체계적으로 갖추는 것이며, 그 중심에 Evaluation Harness가 있다.
평가 대상 파이프라인 각 단계 + Knowledge Graph 품질
  • 컴포넌트 독립 컴포넌트가 교체되어도 평가 코어는 무수정. Evaluator만 유일한 수정 지점.
  • 분산 실행 대응 로컬과 Celery 분산 환경 모두에서 일관 동작.
  • 장기 확장 개발 도구에서 출발, 운영 관측·감사 대시보드의 기반으로 확장.

04 추후 과제
평가 체계 상세화
  • YAML 필드 명세 (depth별 평가 분기, 가중치 정의 등)
  • 그래프 평가 구체화 (golden 형태, entity 매칭 전략)
  • 회귀 판정 상세 정책 (baseline 저장, 임계값, 통계 검정)
운영 연동
  • CI/CD 도구 선정 및 연동
  • Golden 관리 체계 (작성→리뷰→승인 워크플로)
  • 통합 인터페이스 (대시보드 UI, Claude Code 연동 등)
파이프라인 연동
  • 실행 메타 저장 경로 확정 (소요시간, 토큰 수)
  • 병렬 실행 구간 실패 정책

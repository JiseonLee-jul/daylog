# DSPy

LLM 기반 파이프라인을 **프로그래밍 방식으로 선언**하고, 프롬프트를 수동으로 튜닝하는 대신 **자동 최적화**하는 Python 프레임워크. 전통적인 "프롬프트 엔지니어링"을 "LM 프로그래밍"으로 전환시키는 접근으로, 프롬프트 문구를 하나하나 손보는 대신 시스템 아키텍처를 선언하고 옵티마이저가 프롬프트를 자동 생성·개선한다.

## Key Points

- **프로그래밍 패러다임**: 파이프라인을 모듈(Signature, Module)로 선언하고, 옵티마이저가 학습 데이터와 메트릭을 보고 프롬프트를 자동 생성
- **수동 튜닝 대체**: "프롬프트를 조금 다르게 쓰면 결과가 흔들리는" 문제를 아키텍처 레벨에서 해결
- **최적화 자동화**: 사람이 프롬프트를 다듬는 작업을 옵티마이저에 위임 — 지속 가능한 유지보수
- **에이전트 지원**: ReAct 같은 에이전트 패턴을 선언적으로 구성 가능, 도구 사용도 추상화
- **파이프라인 모듈화**: 파이프라인 구성 요소를 재사용 가능한 단위로 분리해 독립적으로 최적화

## Shopify 사례에서의 역할

Shopify의 데이터 구조화 솔루션 진화에서 DSPy는 **2단계 전환의 핵심**이었다. One-Shot LLM(프롬프트에 전체 페이지를 통째로 넘김)에서 DSPy 기반 ReAct 에이전트(직접 브라우징하고 조사하는 도구 사용)로 전환하면서, 프롬프트를 사람이 수동으로 다듬는 대신 **DSPy 옵티마이저가 프로그래밍 방식으로 프롬프트를 자동 최적화**했다. 이후 3단계에서 전문화된 서브 에이전트로 분리될 때도 각 에이전트를 독립적으로 최적화할 수 있게 해준 게 DSPy의 모듈형 구조였다.

결과적으로 Agentic + DSPy + Qwen 조합은 One-Shot GPT-5 대비 비용 1/75, 품질 2배를 달성했다. 핵심 교훈: "프롬프트 문구를 하나하나 손보는 것보다, 시스템 아키텍처를 잘 설계하고 최적화를 자동화하는 편이 지속 가능하다."

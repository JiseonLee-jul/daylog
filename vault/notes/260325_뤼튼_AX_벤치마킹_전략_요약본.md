# 우리의 AX 벤치마킹 전략 요약

  

*뤼튼 AX Report + Agentic AI 발표 자료 기반*

  

---

  

## 왜 이 전략인가

  

뤼튼은 자사 4개 부서에 AI 에이전트를 직접 적용하여 최대 73%의 투입 공수 감소를 실증했고, 이 경험을 패키지(교육+컨설팅+구축)로 묶어 삼성, CJ, KT, LG 등 대기업에 B2B 사업을 전개하고 있다. 동시에, 다수의 기업 AX 프로젝트에서 **성공의 3가지 조건과 실패의 5가지 원인**을 관찰했다. 우리는 이 검증된 프레임을 전략의 뼈대로 삼는다.

  

## 전략의 프레임

  

뤼튼이 정의한 성공 3요소는 "확보해야 할 것", 실패 5요소는 "전 과정에서 피해야 할 것"이다. 이하 모든 전략은 이 프레임 안에서 각 요소를 확보하거나 회피하기 위한 구체적 수단이다.

  

```

┌─ 성공 3요소 ───────────────────────────────────────────────────┐

│                                                                │

│  [병렬 A] 병목 정의          [병렬 B] 기술력                    │

│  Step 1. 적합성 판단         필수 기술 체크리스트 + 현황 평가    │

│  Step 2. 병목 진단           항목별 고도화 전략 수립             │

│         ↓                          ↓                           │

│         └──────────┬───────────────┘                           │

│  Step 3. 3-Layer 분류 → Step 4. 구축 + 측정 (Dogfooding)       │

│                    ↓                                           │

│  [요소3] 경영진 설득: 성과 수치 → Quick Win → 다음 투자 정당화  │

│                                                                │

├─ 실패 5요소 (가드레일) ────────────────────────────────────────┤

│  Quality / Price / Data Infra / UX / AI Literacy               │

└────────────────────────────────────────────────────────────────┘

         ↓

  Dogfooding으로 전 요소의 경험 축적 → 패키지형 B2B 사업화

```

## 성공 요소 1. 생산성 병목 정의 — AX 도입 프로세스

  

뤼튼이 관찰한 기업 사례를 분석하면 AX가 실패하는 가장 흔한 원인은 진단 없이 구축부터 시작하는 것이다. Step 1~2를 건너뛰면 안 된다.

  

**Step 1. 적합성 판단** — 모든 업무에 AI를 적용할 수 있는 것은 아니다. 인터뷰로 수집한 업무를 4가지 조건(명시적 정의 가능성 / 예시 기반 학습 가능성 / 피드백 루프 속도 / 측정 가능성)으로 필터링한다. 이 기준은 이후 B2B 컨설팅의 첫 번째 진단 도구로 그대로 쓰인다.

  

**Step 2. 병목 진단** — 필터를 통과한 업무의 현재 처리 시간을 측정해 기준선을 잡는다. "규칙이 보이고, 평가가 빠르고, 측정이 되고, 사람 의존성이 과도하지 않은 업무"가 Layer 1이자 첫 번째 타겟이다. 이 과정에서 축적한 인터뷰 데이터와 병목 분석 결과가 B2B 워크숍·컨설팅의 실사례 콘텐츠가 된다.

  

**Step 3. 3-Layer 분류** — 업무를 3계층으로 나눈다. 뤼튼 분류표와 대조하되, 병렬 트랙 B(기술력 진단)의 결과가 여기서 합류하여 "기술적으로도 실행 가능한" 현실적 로드맵이 완성된다. 이 분류표가 고객사에 적용할 방법론 템플릿이 된다.

  

- **Layer 1** — 의사결정이 필요 없는 업무 (규칙 반복, 정형 처리)

- **Layer 2** — 데이터 분석은 필요하지만 최종 판단은 아닌 업무

- **Layer 3** — 최종 의사결정이나 외부 시스템 실행이 필요한 업무

  

**Step 4. 구축 + 측정** — Layer 1부터 구축하고, 도입 전후 성과를 측정하며, 트러블슈팅 과정을 문서화한다. Layer 1의 성과 수치가 Layer 2 투자의 승인 근거다. 뤼튼은 고객 응대 Layer 1만으로 건당 처리 시간 19분→3분(-85%)을 달성했고, 이 숫자가 나머지 계층 투자를 정당화했다. 이 전체 과정을 성과 보고서로 정리하면 그것이 곧 B2B 제안서다.

## 성공 요소 2. 기술력 — 필수 기술 체크리스트와 고도화

  

병목 진단과 병렬로, Agent 플랫폼 사업에 필요한 기술 역량을 점검한다. 필수 기술 체크리스트를 만든 후 각 항목의 현재 수준을 평가하고, 자체 구축할 기술과 외부 도구로 해결할 기술의 경계를 명확히 설정한 뒤 항목별 고도화 전략/로드맵을 수립한다. 우리의 기술 현황을 정확히 파악하는 것은 AX 전환 로드맵의 든든한 밑바탕이 된다.

  

- **필수 기술 체크리스트 예시:**

  - RAG (Retrieval-Augmented Generation) — 문서 기반 응답 생성

  - Multi-Agent Architecture — 복수 에이전트 간 협업 및 워크플로우

  - Model Orchestration — 작업 특성에 따른 최적 모델 자동 배치

  - sLLM Fine-tuning — 도메인 특화 소형 모델 학습

  - Tool Calling / MCP — 외부 시스템(API, DB 등) 연동

  - Vector DB & Semantic Search — 유사도 기반 정보 검색

  - 모델 평가 방법론 — 모델별 강점 프로파일링 및 품질 측정

- **모델 스펙 정리표 구축:**

  - 외부 API 모델(Claude, GPT 등)과 로컬 모델(오픈소스 sLLM 등)을 분리

  - 각 모델의 성능 / 비용 / 보안 / 속도 스펙을 비교 정리

  - 고객사 상황에 맞는 모델을 추천하는 능력 자체가 컨설팅의 기술 역량

  - 이 표는 실패 요소 Quality와 Price 대응의 기반 도구가 됨

## 성공 요소 3. 경영진의 의지와 이해 — 성과 측정 체계

  

경영진을 설득하고 지속적 투자를 이끌어내려면, 성과를 "체감상 좋아졌다"가 아닌 숫자로 말해야 한다. 이 숫자는 내부적으로 경영진 설득의 근거이자 외부적으로 B2B 영업의 무기가 된다.

  

- **투입 공수** — 업무별 처리 시간 도입 전후 비교. "월간 1,543분→423분, 73% 감소." 지금 바로 시작 가능.

- **생산성 지수** — 뤼튼 AI Productivity Index를 차용해 단일 숫자로 표현. 공식의 정교함보다 "수학적 모델로 측정한다"는 사실 자체가 신뢰를 준다.

- **모델 평가 방법론(중장기)** — KLAUD를 참고해 "왜 이 업무에 이 모델을 배치했는가"를 설명할 수 있는 기준을 만든다. 그 자체가 컨설팅 상품.

  

Layer 1 Quick Win과 결합하면: 빠르게 성과 수치를 만들고 → 경영진에게 보고하고 → 다음 Layer 투자를 승인받는 사이클이 돌아간다.

## 실패 5요소 가드레일

  

**Quality — 모델 선택 역량을 갖춘다.** "어떤 모델이 좋다"가 아니라 "이 상황에는 이 모델이 최적이다"를 근거와 함께 제시하는 역량이 핵심이다.

  

- 외부 API / 로컬 모델 스펙 정리표를 구축하고 고객사 상황(보안/성능/비용)별 최적 모델을 추천하는 기준 마련

- 뤼튼 KLAUD를 참고해 모델별 강점을 프로파일링하는 평가 방법론을 우리 도메인에 맞게 설계

- 평가 결과를 기반으로 작업 특성별 모델 배치 가이드 수립

  

**Price — 비용 제약 안에서 최대 성과를 설계한다.** 비용 제한은 고객사 환경 제약의 가장 큰 변수다.

  

- 프로젝트 시작 시 고객사 비용 제약을 먼저 파악 후 모델 매핑 스펙 정리표 기반으로 비용 범위 안에서 최적 조합을 설계하는 컨설팅 전략

- 성과 측정 체계를 "이 비용을 써서 이만큼 줄었다"는 비용 정당화 근거로 활용

- Model Orchestration(작업별 모델 분배), Caching(반복 요청 재활용) 등 주어진 환경 내에서의 비용 절감 기술 연구

  

**Data Infra — 자사 데이터 정리 경험을 컨설팅 상품으로 만든다.** 우리 회사의 데이터 환경도 정리가 필요한 상태다. 이를 역으로 활용한다.

  

- 자사 데이터를 정리하며 "어떤 순서로, 어떤 기준으로 정비해야 하는가"의 가이드라인 축적

- 이 가이드라인이 데이터 환경 미비 고객사 대상 컨설팅 상품이 됨

- Dogfooding과 동일한 논리: 자사 경험 → 노하우 → B2B 자산화

  

**UX — 선택지를 제공하고, 피드백 체계를 갖춘다.** "만들었는데 안 쓰는" 상황을 구조적으로 방지한다.

  

- 기존 도구(Slack 등) 활용 / 신규 전용 UX 중 고객사에 선택지 제공, 양쪽 모두 구현

- 사용자 요구사항을 좋은 UX로 변환하는 설계 노하우, 불만 해결 케이스를 체계적으로 축적

- A/B 테스트 체계 구축, 사용자 피드백을 빠르게 수집하고 반영하는 프로세스 확보

  

**AI Literacy — 교육 프로그램을 설계하고, CX Agent로 확장한다.** 교육을 받아도 실무에서 질문은 계속 생긴다. 교육과 상시 지원을 이어서 설계한다.

  

- 개발자/비개발자 역할과 수준에 따른 레벨별 교육 프로그램 설계

- 교육의 확장으로 CX Agent 구축: 제품 사용 문제 대응 + AI 활용 범용 질문 대응

- CX Agent에 들어오는 질문 패턴이 교육 프로그램 개선의 피드백으로 순환

## 횡단 전략

  

### Dogfooding — 모든 전략의 출발점

  

기술은 이미 상향평준화되었다. 뤼튼이 AX 5대 원칙에서 못 박은 것처럼, 차이를 만드는 유일한 요소는 "수많은 유즈케이스를 직접 대응하며 트러블슈팅해본 실전 경험"이다. 자사에 먼저 적용해서 세 가지를 동시에 축적한다:

  

- **영업 신뢰 자산** — "우리가 적용해보니 73% 줄었습니다." 성과 보고서가 곧 B2B 제안서.

- **트러블슈팅 노하우** — AI 오답 처리, 시스템 연동, 데이터 품질, 도입 저항, 비용 최적화. 이 경험이 컨설팅의 핵심 가치.

- **제품 개선 루프** — 자사 적용 → 문제 발견 → 개선 → 고객사 제공 → 새로운 경험 → 다시 제품 반영. 이 루프의 회전 속도가 경쟁력.

  

Dogfooding 과정에서 의도적으로 축적해야 할 자산:

  

| 자산 유형 | 예시 | B2B 활용 |

|---------|------|---------|

| **트러블슈팅 케이스** | AI 오답 발생 시 신뢰도 임계값 조정, 오류 시 대체 경로 설계 | 컨설팅 시 "이런 문제는 이렇게 해결합니다" |

| **UX 실패/성공 사례** | 구성원이 도구를 안 쓰는 문제를 어떻게 극복했는가 | 도입 저항을 줄이는 UX 설계 노하우 |

| **비용 최적화 노하우** | 작업별 최적 모델 배치로 비용 절감한 사례 | 비현실적 기대에 대한 현실적 대안 제시 |

| **도입 저항 관리** | Layer 1부터 점진적 도입으로 저항을 최소화한 과정 | 조직 변화관리 컨설팅 역량 |

| **데이터 품질 대응** | 데이터가 정리 안 된 상태에서 어떻게 시작했는가 | 데이터 환경 미비 기업 대상 솔루션 |

  

### 패키지형 B2B — 묶어서 판다

  

도구만 납품하면 안 쓰이고(H사), 진단 없이 구축하면 방향이 틀리고(S사), 구축 없이 컨설팅만 하면 실행이 안 된다(A사). 교육 + 컨설팅 + 구축 + CX를 묶는다.

  

| 패키지 | 내용 | Dogfooding에서 오는 것 |

|--------|------|----------------------|

| **교육** | 레벨별 AI 이해도 교육 + 경영진 정렬 | 레벨별 커리큘럼, 도입 저항 극복 사례 |

| **컨설팅** | 적합성 판단 → 3-Layer 분류 → 로드맵, 데이터 진단, 모델 추천 | 필터링 기준, 분류 템플릿, 모델 매핑표, 데이터 정리 가이드 |

| **구축** | Layer 1 시범 → 성과 검증 → Layer 2~3 확장 | 트러블슈팅 케이스, 비용 최적화, 구축 속도 |

| **CX** | 제품 문제 대응 + AI 활용 질문 대응 | CX Agent 구축 경험, 질문→교육 개선 루프 |

  

**수익 흐름:** 교육(진입) → 컨설팅(진단) → Layer 1 구축(성과 입증) → Layer 2~3 확장(업셀) → CX + 유지보수(정기 수익)

### 실행 1. 캐치업 — 플랫폼 필수 기능 확보

  

성공 요소 2(기술력)의 체크리스트가 "알아야 할 기술"이라면, 아래는 그 기술이 **제품으로 구현될 때의 형태**다. 리튼, 사이오닉, 업스테이지, Dify 등 선발 주자들이 이미 기본으로 갖추고 있는 기능이며, 이것 없이는 경쟁 테이블에 앉을 수 없다.

  

**플랫폼 필수 기능 후보 예시:**

- 에이전트 등록/관리 UI (드래그 앤 드롭)

- 자연어 기반 에이전트 생성

- LLM 멀티 모델 교체 지원

- 프롬프트 실험 환경

- RBAC (역할 기반 접근 제어)

- 대시보드 / 모니터링

- 문서 파싱 (Document Parse)

- RAG 파이프라인

- CI/CD, 클라우드 운용, SaaS 제공

  

사업 구조 측면에서도 캐치업이 필요하다. 리튼 모델에서 배울 점:

- 단품 판매가 아닌 **플랫폼 구축 + AX 컨설팅 + 사내 교육/전파** 패키징 판매 전략

- Agent 도입 후 **KPI 설정 및 ROI 측정 체계** 수립

  

**후속 작업:**

1. **시장 조사**: 주요 Agent 플랫폼(리튼, 사이오닉, 업스테이지, Dify, CrewAI 등)의 기능 매트릭스 작성 → 위 목록 확정

2. **기술력 갭 분석**: 성공 요소 2의 체크리스트와 대조하여 항목별 현재 수준 평가 (가능 / 부분 가능 / 미보유)

3. **기술력 향상 로드맵**: 갭이 있는 영역의 학습/개발 계획 수립

4. **서비스 패키지 모델 설계**: 리튼의 패키징 전략을 참고하여 자사 버전 재설계

### 실행 2. 차별화 — 기술 컨셉 벤치마킹

  

필수 기능을 갖추는 것만으로는 차별성이 없다. 경쟁 기업들의 기술 컨셉 중, 우리 포지셔닝에 맞는 것을 선별하여 벤치마킹한다.

  

| 기술 컨셉 | 설명 | 참고 사례 |

|-----------|------|-----------|

| **스무고개식 요구사항 도출** | 막연한 사용자에게 질문을 던져 요구사항을 점진적으로 구체화 | 인핸스 빌드 어플로크 |

| **자연어 ↔ 워크플로우 양방향 변환** | 자연어 입력 → 코드/그래프 동시 생성, 1:1 대응 구조 | 사이오닉 중간 컴파일 |

| **틀 안의 자유 (Constrained Generation)** | 프레임워크/UI 틀은 고정, 로직만 자유 생성 → 품질 일관성 | Lovable (프론트), 오토비 (백엔드) |

| **백엔드 자동 설계 + 샌드박스** | 에이전트 작업 시 격리 환경 자동 생성, 규칙 내 코드 생성 | 리튼 AutoBe |

| **멀티 에이전트 오케스트레이션** | 다중 에이전트 협업으로 복잡한 업무 시나리오 처리 | 리튼 Multi Orchestration |

| **도메인별 에이전트 템플릿화** | 코어 동일, 도메인 로직/데이터 모델만 특화하여 고객별 맞춤 | 인핸스(커머스), 업스테이지(보험/헬스케어/제조/금융) |

  

**후속 작업:**

- 시장 조사를 통한 경쟁 기업의 특장점 분석

- 각 컨셉별 PoC 가능성 검토, 기술 컨셉별 도입 우선순위 확정

- 기능 구현을 위한 논문/기술 자료 조사

### 실행 3. 우리의 포지셔닝 — 벤치마킹 위에 더하는 것

  

캐치업으로 기본기를 갖추고, 벤치마킹으로 고도화 방향을 잡은 뒤, 최종적으로 **우리만이 놓을 수 있는 차별점**이다.

  

**타겟 시장: 후발 사용자**

  

선발 주자(리튼 등)의 고객은 이미 AI를 인지하고 찾아오는 앞선 기업이다. 우리가 같이 가는 고객은 AI를 아직 잘 모르는, 뭐부터 시작할지 모르는 기업이다. 이 시장이 훨씬 크고, 실패 요소 AI Literacy에서 설계한 교육 프로그램이 바로 이 시장의 진입 도구가 된다.

  

**차별점 1. 교육 역량 × 플랫폼의 결합**

- 수많은 교육 진행 경험 + 1,500명 단톡방 네트워크 보유

- 교육으로 고객 유입 → 플랫폼 도입 → 도메인 경험 축적의 선순환 — 횡단 전략 Dogfooding 루프와 직결

- 동일한 기능이라도 후발 사용자 눈높이에 맞춘 설계: 에이전트 빌더 진입 전부터 "본인이 뭘 모르는지도 모르는" 사용자를 안내하는 온보딩

- 구축 후 교육 전파까지 원스톱 — 횡단 전략 패키지형 B2B의 "교육 → 컨설팅 → 구축 → CX" 흐름을 교육 강점으로 강화

## 참고: 영역별 AX 진입 난이도 (뤼튼 실증)

  

- **고객 응대(CX)** — 공수 감소 73%, 생산성 증가 35%. 4개 영역 중 1위. 모든 기업에 존재하고 측정이 쉬움

- **백엔드 > 프론트엔드** — 백엔드 55% vs 프론트 28%. 입출력 정형화 + 자동 검증 가능

- **CI/CD** — 프론트/백엔드 모두 자동화율 85%+. 개발 AX의 가장 쉬운 진입점

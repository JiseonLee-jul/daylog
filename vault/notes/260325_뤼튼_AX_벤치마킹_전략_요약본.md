# 우리의 AX 벤치마킹 전략 요약

*뤼튼 AX Report + Agentic AI 발표 자료 기반*

---

## 전략 흐름

```
자사 적용(Dogfooding) → 성과를 수치로 증명 → 패키지형 B2B 사업화
```

---

## 1. 사내 AX를 이렇게 진행한다

**Step 1~2를 건너뛰고 구축부터 하면 실패한다.** 뤼튼이 관찰한 시장 실패의 공통 원인이다.

**Step 1. 적합성 판단** — 부서별 인터뷰로 업무를 수집한 뒤, 4가지 조건(명시적 정의 가능성 / 예시 기반 학습 가능성 / 피드백 루프 속도 / 측정 가능성)으로 필터링한다. 통과한 업무만 AX 대상. 이 필터링 기준은 이후 B2B 컨설팅의 진단 도구로 그대로 쓴다.

**Step 2. 병목 진단** — 필터를 통과한 업무의 현재 평균 처리 시간을 측정해 기준선을 잡는다. "규칙이 보이고, 평가가 빠르고, 측정이 되고, 사람 의존성이 과도하지 않은 업무"를 우선 대상으로 선정한다. 여기서 축적한 인터뷰 데이터와 병목 분석 결과가 B2B 워크숍/컨설팅의 실사례 콘텐츠가 된다.

**Step 3. 3-Layer 분류** — 업무를 3계층(규칙 반복 / 데이터 판단 / 의사결정/실행)으로 나누되, 뤼튼이 8개 부서x3계층으로 이미 분류한 것과 겹치는 업무는 그대로 차용하고, 우리 고유 업무만 자체 분류한다. 이 분류표가 우리의 AX 로드맵이자, 고객사에 적용할 방법론 템플릿이 된다.

**Step 4. 구축 + 측정** — Layer 1부터 구축하고, 도입 전후 성과를 측정하고, 트러블슈팅 과정을 문서화한다. Layer 1 성과 수치가 Layer 2 투자의 승인 근거다. 이 전체 과정을 성과 보고서로 정리하면 그것이 곧 B2B 제안서다.

## 2. 성과 측정 체계를 갖춘다

**투입 공수** — 업무별 평균 처리 시간을 도입 전후로 비교. "월간 1,543분 → 423분, 73% 감소" 같은 숫자를 만든다. 지금 바로 시작 가능.

**생산성 지수** — 뤼튼의 AI Productivity Index를 차용해 "생산성 35% 증가"라는 단일 숫자로 표현. 수학적 모델로 측정한다는 사실 자체가 신뢰를 준다.

**모델 평가 방법론(중장기)** — 뤼튼의 KLAUD를 참고해 우리 도메인에 맞는 모델 평가 기준을 만든다. "왜 이 업무에 이 모델을 배치했는가"를 설명할 수 있으면, 그 자체가 컨설팅 상품.

## 3. Dogfooding으로 제품과 사업 역량을 동시에 키운다

자사에 먼저 적용해서 세 가지를 축적한다:
- **영업 신뢰 자산** — 자사 성과 보고서가 곧 B2B 제안서
- **트러블슈팅 노하우** — AI 오답 처리, 시스템 연동, 데이터 품질, 도입 저항, 비용 최적화 경험. 기술이 상향평준화된 시장에서 유일한 차별화
- **제품 개선 루프** — 자사 적용 → 문제 발견 → 개선 → 재적용 → 고객사 제공 → 새로운 경험 축적 → 다시 제품 반영. 이 루프의 회전 속도가 경쟁력

## 4. B2B는 패키지로 판다

도구만 납품하면 안 쓰이고, 진단 없이 구축하면 방향이 틀리고, 구축 없이 컨설팅만 하면 실행이 안 된다. 세 가지를 묶는다:

| 패키지 | 내용 | Dogfooding에서 오는 것 |
|--------|------|----------------------|
| **워크숍** | AI 이해도 교육 + 경영진 목표 정렬 | 도입 저항 극복 사례 |
| **컨설팅** | 적합성 판단 → 인터뷰 → 3-Layer 분류 → 로드맵 | 적합성 필터링 기준, 분류 템플릿, "이 업무는 AI에 부적합" 판단 근거 |
| **구축** | Layer 1 시범 → 성과 검증 → Layer 2~3 확장 | 트러블슈팅 케이스, 비용 최적화 노하우, 구축 속도 |

**수익 흐름:** 워크숍(진입) → 컨설팅(진단) → Layer 1 구축(성과 입증) → Layer 2~3 확장(업셀) → 유지보수(정기 수익)

## 5. AX 실패 5요소를 이렇게 피한다

| 실패 요소 | 우리의 대응 |
|---------|----------|
| 품질 | 모델 평가 방법론 차용, 작업별 최적 모델 배치. 비현실적 기대 사전 관리 |
| 비용 | 규칙 기반 처리 가능한 작업은 LLM 미사용. 비용 구조 투명 공개 |
| 데이터 환경 | 자체 데이터 환경 진단 프레임워크를 통해 컨설팅 단계에서 진단. 미비하면 정비부터 |
| UX | Slack 등 기존 도구를 인터페이스로 활용. 새 도구 강제 안 함 |
| AI 이해도 | 교육을 제품에 번들링. 도구만 주면 안 쓴다는 전제로 설계 |

## 참고: 영역별 AX 진입 난이도 (뤼튼 실증)

- **고객 응대(CX)** — 공수 감소 73%, 생산성 증가 35%. 4개 영역 중 1위. 모든 기업에 존재하고 측정이 쉬움
- **백엔드 > 프론트엔드** — 백엔드 55% vs 프론트 28%. 입출력 정형화 + 자동 검증 가능
- **CI/CD** — 프론트/백엔드 모두 자동화율 85%+. 개발 AX의 가장 쉬운 진입점
